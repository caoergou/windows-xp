# AGENTS.md - 设计原则与指导方针

## 核心设计原则

### 1. 忠实还原 Windows XP 经典风格

**最重要的原则**：本项目的目标是**忠实还原 Windows XP 的经典 UI 风格**，而不是创造现代化或改进的版本。

- ✅ 所有 UI 组件应该尽可能接近真实的 Windows XP 外观
- ✅ 参考真实的 Windows XP 截图和设计规范
- ❌ 不要添加现代化的设计元素（圆角、阴影、渐变等，除非 XP 原生就有）
- ❌ 不要使用过于"新"的视觉效果
- ❌ 不要自行"改进"或"美化" XP 的设计

### 2. Internet Explorer 6 风格

IE 浏览器组件必须还原 IE6 的经典外观：

- 菜单栏：文件(F) 编辑(E) 查看(V) 收藏(A) 工具(T) 帮助(H)
- 工具栏：绿色的前进/后退按钮（不是蓝色！）
- 地址栏：经典的 XP 风格输入框
- 状态栏：底部的状态信息栏
- 整体色调：#ECE9D8 背景色（XP 经典灰）

### 3. 桌面图标样式

- 图标阴影：使用 `filter: drop-shadow(2px 2px 3px rgba(0,0,0,0.7))`
- 文字阴影：使用简单的 `text-shadow: 1px 1px 2px rgba(0, 0, 0, 0.8)`
- 不要使用多重阴影或过重的阴影效果
- 文字和图标的阴影应该保持一致的风格

### 4. 快捷方式箭头

- 使用简洁的 SVG 路径绘制弯箭头
- 白色填充，深色描边
- 位于图标右下角
- 路径示例：`M2,11 L2,9 L5,9 L5,2 L3,2 L6,0 L9,2 L7,2 L7,11 Z`

### 5. 窗口管理

- 窗口拖动：使用 `react-draggable` 的 `defaultPosition` 模式
- 窗口定位：wrapper div 必须设置 `top: 0; left: 0` 以避免偏移
- 上下文菜单：使用 `createPortal` 渲染到 body，避免 transform 影响定位

### 6. 文件系统

- 文件结构定义在 `src/data/filesystem.json`
- 回收站项目在 `src/data/recycle_bin/*.json`
- 支持文件拖放、重命名、删除等操作

## 开发注意事项

### 修改 UI 组件时

1. **先查看真实的 Windows XP 截图**
2. 确保颜色、字体、间距与 XP 原版一致
3. 不要添加 XP 中不存在的功能或视觉效果
4. 测试在不同分辨率下的表现

### 常用 XP 颜色

- 窗口背景：`#ECE9D8`
- 标题栏激活：`linear-gradient(to right, #0997FF, #0053EE)`
- 标题栏未激活：`linear-gradient(to right, #7A96DF, #5A7ACF)`
- 按钮边框：`#003C74`
- 菜单高亮：`#316AC5`

### 常用 XP 字体

- 主字体：`Tahoma, "Microsoft YaHei", sans-serif`
- 字号：11px - 12px（大部分 UI）

## 测试清单

在提交修改前，请确保：

- [ ] UI 外观与真实 XP 截图对比无明显差异
- [ ] 所有交互功能正常（点击、拖动、右键菜单等）
- [ ] 窗口定位正确，无偏移问题
- [ ] 拖动操作无残影或视觉错误
- [ ] 在 1024x768 和 1920x1080 分辨率下都能正常显示
