# docs-site

The VitePress documentation site for `@caoergou/windows-xp` (#214), deployed to
GitHub Pages alongside the landing page at **`/windows-xp/docs/`**.

It's a **self-contained workspace** — its own `package.json` and lockfile — so
VitePress's Vue toolchain never touches the engine's install or CI `npm ci`.

## Develop

```bash
cd docs-site
npm install
npm run docs:dev      # local dev server (generates the TypeDoc API first)
```

## Build

```bash
npm run docs:build    # → .vitepress/dist  (runs docs:api, then vitepress build)
```

The root `deploy.yml` builds this after the main Vite site and copies the output
into `dist/docs/`, so Pages serves it at `/windows-xp/docs/`.

## Structure

- `index.md` — home (hero + features), with a `zh/` mirror.
- `guide/*.md` — the guide pages, migrated from the old `USAGE.md` (which is now
  a thin jump index). Each old `USAGE.md` `##` section is one page here.
- `guide/events.md` — the event reference; the table between the
  `<!-- EVENTS:START/END -->` markers is generated by
  `scripts/gen-events-doc.mjs` (root `npm run docs:events`), drift-guarded by
  `test/eventDocs.test.ts`.
- `components.md` — embeds the live component gallery (`/windows-xp/gallery/`).
- `api/` — TypeDoc markdown generated from `../src/lib` (git-ignored, built on
  demand via `npm run docs:api`).

Code examples on every page are import-checked by `test/doc-examples.test.ts`, so
a page can never advertise an export that doesn't exist.
