# 《山月无声》解密与引导优化建议 2.0

## 问题分析

之前的设计方案存在以下问题：
1. 过于自由，缺乏明确的引导，导致玩家认知进度不可控
2. 信息获取顺序无约束，重要线索可能被玩家错过或提前看到
3. 缺少对玩家理解程度的判断，无法确保玩家真正理解关键信息
4. 玩家容易陷入探索疲劳，无法明确下一步行动方向

## 优化方案：有控制的自由探索系统

### 核心思路

采用 **"线性引导 + 局部自由"** 的系统，既能保证玩家按正确的节奏获取信息，又保留一定的自由探索空间。

**关键创新：认知节点系统**
- 每个节点代表玩家对案件的一个关键理解层次
- 节点状态：未访问 → 已接触 → 已理解（解锁下一节点）
- 触发条件：玩家必须在当前节点完成指定的"关键行为"

### 触发机制实现

#### 关键行为检测
- **阅读消息**：在群聊或私聊中停留超过3秒（防止快速划过）
- **查看页面**：在QZone、贴吧、邮件中停留超过5秒
- **解锁内容**：输入正确密码并成功解锁
- **交互操作**：选中文字、右键属性、搜索特定关键词

#### 访问计数系统
每个节点都有一个"访问计数"，当玩家完成关键行为时计数+1，达到阈值后自动解锁下一节点。

### 引导系统

#### 隐性引导（不打破沉浸感）
1. **调查笔记**：在关键节点自动弹出，总结当前发现，暗示下一步
2. **QQ状态提示**：林晓宇头像旁显示"最后登录时间2016.02.14"
3. **文件管理器高亮**：D盘的"给小灯"文件夹在解锁其他内容后更亮
4. **邮件提醒**：新邮件到达时，任务栏图标闪烁

#### 显性引导（仅在玩家卡住时出现）
1. **提示计数器**：连续5分钟无操作 → 弹出提示窗口
2. **智能提示**：根据当前节点状态给出针对性提示（如"试试在群聊中搜索'光之公式'"）
3. **跳过选项**：尝试密码错误5次后 → 给出提示；错误10次后 → 提供跳过选项

### 信息屏蔽与节点锁定

#### 信息屏蔽
- 加密相册中的照片在解锁前只显示缩略图
- 加密日志的预览区文字未选中时不可见
- 回收站中的文件在解锁前不可恢复

#### 节点锁定
- 加密相册密码只有在访问节点2.1和2.2后才会显示正确提示
- 加密日志密码只有在解锁相册后才会显示提示
- 父亲第二层日志密码只有在解锁加密日志后才会显示提示

5. **`src/apps/Explorer.jsx`**：添加访问计数逻辑

1. **`useNodeTracker` 钩子**：用于跟踪节点访问计数
2. **`useStuckDetector` 钩子**：用于检测玩家是否卡住
3. **`SmartHint` 组件**：根据节点状态显示智能提示

### 流程示例

**玩家行为路径**：
打开电脑 → 点击便签（节点1.1）→ 打开QQ（节点1.2）→ 看群聊（节点1.3）→ 访问林晓宇空间（节点1.4）→ 发现光之公式（节点2.1）→ 解锁给小灯文件夹（节点2.2）→ 阅读父亲日志（节点2.3）→ 解锁加密相册（节点2.4）→ 发现隐写文字（节点3.1）→ 解锁加密日志（节点3.2）→ 理解陈默动机（节点3.3）→ 解锁父亲第二层日志（节点4.1）→ 阅读完整信件（节点4.2）→ 游戏完成（节点4.3）

## 预期效果

通过以上优化，玩家将获得以下体验：
1. **明确的方向感**：知道在每个阶段应该做什么
2. **可控的认知进度**：确保玩家按正确的顺序理解关键信息
3. **避免探索疲劳**：通过引导系统防止玩家陷入迷茫
4. **增强沉浸感**：隐性引导不打破游戏氛围
5. **平衡自由度**：局部自由探索 + 整体线性引导

## 风险控制

### 节点跳转保护
- 玩家只能解锁下一节点，不能跳过中间节点
- 节点状态在localStorage中持久化，防止刷新后进度丢失

### 容错机制
- 对于关键节点，提供多种触发方式
- 允许玩家通过多种路径解锁同一节点
- 对于难点节点，提供渐进式提示

### 进度重置
- 提供完整的进度重置功能，允许玩家重新开始游戏
- 保留已解锁的节点状态，但允许重新触发

## 总结

本优化方案通过认知节点系统实现了对玩家探索过程的有效控制，同时保留了一定的自由探索空间。这种设计既保证了玩家按正确的节奏获取信息，又避免了线性引导的单调性，提供了更加沉浸式的解谜体验。
