## 📖 二、游戏设计目标理解

### 2.1 故事核心

**类型**：社会派悬疑互动解谜游戏

**主题**：当制度的缝隙吞噬个体，真相便沉没于系统的沉默与个体的怯懦之中

**玩家角色**：2026年的夏灯，通过父亲遗留的XP电脑调查2015-2016年的悲剧真相

**游戏时长**：3-4小时（180-240分钟）

### 2.2 核心人物

1. **林晓宇**（2000-2016）- 摄影少年，真相记录者，象征"无声"
2. **陈默**（1999-）- 技术少年，被迫参与替考，象征"山"
3. **夏灯**（1999-）- 教导主任之女，玩家角色，象征"灯"
4. **夏建国**（1965-2025）- 县一中教导主任，知情但沉默的记录者
5. **张雨**（1998-）- 林晓宇暗恋的学姐，高考移民受害者

### 2.3 四阶段叙事结构

```
阶段一：个体悬疑 (60分钟)
├─ 问题：林晓宇是怎么死的？
├─ 情感：怀旧、困惑
└─ 认知：意外坠亡 → 林发现了什么？

阶段二：证据指向 (50分钟)
├─ 问题：陈默是凶手吗？
├─ 情感：愤怒、背叛
└─ 认知：陈默至少是共犯

阶段三：真相反转 (40分钟)
├─ 问题：为什么会这样？
├─ 情感：震撼、理解
└─ 认知：陈默也是受害者

阶段四：系统揭露 (30分钟)
├─ 问题：这背后是什么？
├─ 情感：沉重、反思
└─ 认知：系统性腐败（高考移民产业链）
```

## 💡 关键建议

### 技术实现建议

1. **解密系统**
   - 使用 ModalContext 统一管理密码输入弹窗
   - 密码验证逻辑集中在 Context 层
   - 支持密码提示和错误次数限制

2. **数据组织**
   - 使用 `import.meta.glob` 动态加载内容
   - JSON 文件保持扁平化，避免过度嵌套
   - EXIF 数据单独存储为 `{filename}_exif.json`

3. **性能优化**
   - 大量聊天记录使用虚拟滚动
   - 照片使用懒加载
   - 考虑使用 React.memo 优化渲染

### 内容制作建议

1. **说说内容**
   - 参考真实的2015-2016年QQ空间风格
   - 使用当时流行的网络用语
   - 时间戳要符合学生作息规律

2. **聊天记录**
   - 对话要符合高中生语气
   - 技术讨论要真实可信
   - 情感变化要渐进自然

3. **照片EXIF**
   - 使用真实的相机参数
   - 时间戳要与故事时间线一致
   - 曝光三角密码要合理（光圈2.8、快门1/250、ISO400）

### 测试要点

1. **解密流程**
   - 每个密码都能正确解锁
   - 错误密码有明确提示
   - 解锁后内容正确显示

2. **内容完整性**
   - 所有时间线节点都有对应内容
   - 人物性格前后一致
   - 线索链条完整

3. **游戏体验**
   - 3-4小时能完整通关
   - 解密难度适中
   - 情感递进自然
