# 提示系统测试指南

## 系统概述

提示系统已完全集成到游戏中，为5个核心谜题提供渐进式提示：

1. **QQ密码** (`qq_password`)
2. **相册密码** (`album_password`)
3. **父亲日志第一层** (`father_log_layer1`)
4. **林晓宇加密日志** (`encrypted_diary`)
5. **父亲日志第二层** (`father_log_layer2`)

## 提示触发机制

提示系统基于玩家的错误尝试次数自动解锁：

- **5次错误** → 方向性提示（告诉玩家应该往哪个方向思考）
- **10次错误** → 详细线索位置（告诉玩家具体在哪里找线索）
- **15次错误** → 半答案（给出部分答案或格式提示）
- **20次错误** → 完整答案（直接给出密码）

## UI 元素说明

### 密码对话框新增元素

1. **尝试计数器**: 显示"已尝试 X 次"
2. **提示按钮**: 黄色背景的"💡 提示"按钮
   - 未解锁时显示"再尝试 X 次解锁提示"
   - 解锁后可点击查看提示
3. **跳过按钮**: 红色背景（仅辅助谜题，当前未启用）
4. **提示框**: 黄色背景框显示提示内容

### 提示框样式

- 浅黄色背景 (`#FFFEF0`)
- 金色边框 (`#FFB900`)
- 标题加粗，橙色文字
- 内容黑色文字，易读

## 数据持久化

所有尝试次数保存在 `localStorage` 的 `xp_game_progress` 中：

```json
{
  "puzzleAttempts": {
    "qq_password": 5,
    "album_password": 12,
    "father_log_layer1": 3
  }
}
```

### 重置测试数据

如需重置某个谜题的尝试次数，在浏览器控制台执行：

```javascript
// 查看当前进度
let progress = JSON.parse(localStorage.getItem('xp_game_progress'));
console.log(progress.puzzleAttempts);

// 重置特定谜题
progress.puzzleAttempts.qq_password = 0;
localStorage.setItem('xp_game_progress', JSON.stringify(progress));

// 或完全重置所有进度
localStorage.removeItem('xp_game_progress');
location.reload();
```

## 配置文件

### 提示内容配置

文件: [src/data/puzzle_hints.json](../../src/data/puzzle_hints.json)

每个谜题包含：
- `name`: 谜题名称
- `type`: 类型（main/auxiliary）
- `hints`: 提示数组
  - `threshold`: 触发阈值
  - `level`: 提示级别
  - `title`: 提示标题
  - `content`: 提示内容

### 谜题映射配置

文件: [src/utils/puzzleMapping.js](../../src/utils/puzzleMapping.js)

定义了如何将文件路径、相册名称等映射到 puzzleId。

## 已知问题和注意事项

1. **密码大小写敏感**: 确保测试时使用正确的大小写
2. **刷新页面**: 尝试次数会持久化，刷新页面不会重置
3. **多个窗口**: 如果同时打开多个密码对话框，它们共享同一个计数器

## 扩展提示系统

### 添加新谜题

1. 在 `puzzle_hints.json` 中添加新谜题配置
2. 在 `puzzleMapping.js` 中添加映射规则
3. 确保调用 `showPasswordDialog` 时传递正确的 `puzzleId`

### 修改提示阈值

编辑 `puzzle_hints.json` 中的 `threshold` 值即可。

### 添加辅助谜题（可跳过）

1. 在 `puzzleMapping.js` 的 `isAuxiliaryPuzzle()` 中添加谜题ID
2. 玩家尝试10次后会显示"跳过"按钮
