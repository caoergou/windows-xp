# 《山月无声》一致性问题报告

*审查范围：所有核心设计文档 + 辅助文档 + src/data/ 游戏数据抽样*

## 一、阻断性问题（会导致游戏流程卡死或严重混乱）

### 1.1 高考移民人数矛盾：15人 vs 18人 vs 6人

各文档对高考移民人数的描述严重不一致：

| 文档 | 描述 |
|------|------|
| CLAUDE.md | "15人" |
| 大纲.md | "2013-2015年，3年共操作18人（每年6人）" |
| 解密.md（新闻报道段） | "18人、三年、2000万" |
| 设计.md（关键词过滤规则） | "15人"（禁用词列表） |
| 设计.md（信息分阶段释放） | "产业链规模（18人）" |
| 设计.md（记者手记） | "该县共有18人...涉案金额约2000万元" |
| 大纲.md（阶段二隐藏） | "15人、三年、250万" |

**结论**：主体叙事已统一为"18人"，但 CLAUDE.md 和设计.md 的禁用词列表仍残留"15人"，大纲.md 阶段二隐藏列表中出现"250万"这个在其他任何地方都不存在的数字。需要统一为 **18人**，并清理所有残留。

### 1.2 涉案金额矛盾：1800万 vs 2000万 vs 250万

| 文档 | 描述 |
|------|------|
| 大纲.md（操作流程段） | "总收益约1800-2000万" |
| 大纲.md（阶段二隐藏） | "250万"（禁用词） |
| 解密.md（新闻报道） | "2000万" |
| 设计.md（记者手记） | "涉案金额约2000万元" |
| 设计.md（禁用词列表） | "250万" |

**结论**："250万"完全无法解释来源，可能是早期版本残留。"1800-2000万"和"2000万"基本一致，但需要统一口径。建议统一为 **2000万**。

### 1.3 父亲文件夹密码与结构矛盾

**设计.md** 定义的父亲文件夹结构：

```
D:\给小灯\ (密码: 19990315)
  ├─ README.txt
  ├─ 工作日志2015-2025.txt
  ├─ 网页截图\
  └─ 爸爸的信\ (密码: 20151123)
      └─ 给夏灯.txt
```

**解密.md** 定义的结构：

- 第一层（密码19990315）：包含"我的懦弱.txt"和"police_record_copy.txt"
- 第二层（密码20151123）：包含《给夏灯.txt》

**filesystem.json** 实际结构：

- 第一层（密码19990315）：包含 README.txt、工作日志文件夹（按年月组织的日记）、网页截图、爸爸的信文件夹
- 第二层（密码20151123）：只有"给夏灯.txt"
- **没有** "我的懦弱.txt" 和 "police_record_copy.txt"

**结论**：解密.md 提到的"我的懦弱.txt"和"police_record_copy.txt"在 filesystem.json 中不存在。实际实现用的是按日期组织的工作日志文件夹。需要决定以哪个为准并统一。

### 1.4 项目现状文档的密码与谜题编号严重偏离

**项目现状与实施计划.md** 列出7个谜题，密码与解密.md 完全不同：

| 谜题 | 项目现状.md | 解密.md |
|------|-----------|---------|
| 父亲文件夹第一层密码 | `20151123` | `19990315` |
| 父亲文件夹第二层密码 | `20160215` | `20151123` |
| 谜题总数 | 7个 | 5个 |
| 谜题顺序 | QQ登录→相册提示→相册→父一层→父二层→日志→诗 | 相册→父一层→日志→诗→父二层 |

**结论**：项目现状.md 的密码体系与解密.md/设计.md/filesystem.json 完全矛盾。这是最严重的阻断性问题之一，因为开发者如果参考项目现状.md 会实现错误的密码。**应以解密.md + filesystem.json 为准**。

### 1.5 电脑登录密码矛盾

| 来源 | 密码 |
|------|------|
| user_config.json（实际数据） | `shanyue2015` |
| 解密.md | "密码是你生日"（暗示19990315） |
| 设计.md（密码列表） | `19990315`（QQ登录 + 父亲文件夹第一层共用） |

**结论**：实际游戏数据中电脑登录密码是 `shanyue2015`，但设计文档说"密码是你生日"。这两者不一致。需要明确：电脑登录密码和QQ登录密码是否相同？如果不同，玩家如何获知电脑密码？

## 二、混淆性问题（不会卡死但会让玩家/开发者困惑）

### 2.1 游戏总时长矛盾

| 文档 | 总时长 | 解密时长 | 阅读时长 |
|------|--------|---------|---------|
| 大纲.md | 3.5-4小时 | - | - |
| 解密.md | 180分钟（3小时） | 60分钟 | 120分钟 |
| 设计.md | 180分钟（3小时） | 75分钟 | 105分钟 |
| 项目现状.md | 3-4小时 | - | - |
| CLAUDE.md | 3小时 | - | - |

**结论**：大纲.md 说3.5-4小时，其他文档说3小时。解密/阅读时长分配也不一致（60/120 vs 75/105）。建议统一。

### 2.2 四阶段时长分配矛盾

| 阶段 | 解密.md | CLAUDE.md | 设计.md（信息释放表） |
|------|---------|-----------|---------------------|
| 阶段一 | 60分钟 | 60分钟 | 0-60分钟 |
| 阶段二 | 50分钟 | 70分钟 | 60-110分钟 |
| 阶段三 | 40分钟 | 50分钟 | 110-150分钟 |
| 阶段四 | 30分钟 | 40分钟 | 150-180分钟 |
| 总计 | 180分钟 | 220分钟 | 180分钟 |

**结论**：CLAUDE.md 的四阶段时长加起来是220分钟，超过了所有文档声称的180分钟总时长。CLAUDE.md 的阶段时长与解密.md 也不一致。

### 2.3 陈默邮件数量矛盾

| 文档 | 邮件数量 |
|------|---------|
| 设计.md（邮件触发表） | 6封 |
| 设计.md（素材优先级） | "陈默邮件6封" |
| 解密.md | 提到阶段一、二、三各有邮件，但未明确总数 |
| chenmo_correspondence.json（实际数据） | 6封（email_chenmo_001 到 006） |

**结论**：邮件数量基本一致（6封），但设计.md 的触发条件表和实际数据的 trigger 字段不完全匹配。设计.md 说邮件2触发条件是"玩家回复邮件1"，但实际数据中 trigger 是 `player_view_qzone`。

### 2.4 陈默邮件地址矛盾

### 2.5 红鲱鱼编号矛盾

### 2.6 林晓宇QQ空间个人说明矛盾

### 2.9 林晓宇加密日志中的日期矛盾

## 三、细节性问题（不影响游戏但需要注意）

### 3.1 林晓宇年龄/年级矛盾

| 文档 | 描述 |
|------|------|
| 人物.md | 2000年出生，2015年15岁 |
| 时间线.md | 2014年9月1日成为高一新生 |
| 大纲.md | 2015年9月"山顶约定" |

**结论**：如果2000年出生、2014年9月入学高一，则入学时14岁（正常）。但人物.md 说"2015年15岁"，这意味着2000年出生的人在2015年确实是15岁（如果生日在事件之前）。时间线一致。但大纲.md 说"山顶约定"在2015年9月，而时间线.md 说山顶事务所成立于2014年9月15日。大纲.md 的时间有误。

### 3.2 夏灯年级矛盾

群聊数据（mountain_office.json）中，2015年9月1日夏灯说"我被分到快班了"。

但人物.md 说夏灯是"县一中高一'清北快班'学生"（2015年）。如果2014年入学高一，2015年应该是高二。人物.md 的"高一"描述有误，应为"高二"。

### 3.3 张雨年级时间线问题

| 文档 | 描述 |
|------|------|
| 大纲.md | 张雨(1998-)，林晓宇暗恋的学姐 |
| 群聊数据 | 2014年12月20日林晓宇说"今天在图书馆看到高二的张雨学姐" |
| 时间线.md | 2015年6月高考 |

**结论**：如果2014年12月张雨是高二，那2015年6月她应该是高二下学期，不是高三，不应该参加高考。张雨应该在2014年12月是高三才对。群聊数据中"高二的张雨学姐"应改为"高三的张雨学姐"。

### 3.4 "光之公式"首次提及时间矛盾

| 来源 | 时间 |
|------|------|
| 时间线.md | 2015年8月12日"林晓宇在群里提到'光之公式'" |
| 设计.md（群聊数据） | 2015年11月20日"光之公式"对话 |
| 解密.md | 群聊记录（2015-11-20）提及相机设置 |
| 说说数据 | 2015年8月12日说说提到"光之公式" |
| 群聊数据 | 2015年11月20日才在群聊中详细讨论参数 |

**结论**：时间线.md 和说说数据说8月12日就提到了"光之公式"，但群聊中详细讨论参数是11月20日。这不矛盾（可以先在说说提到概念，后在群聊详细讨论），但解密.md 的线索来源只指向11月20日群聊，如果玩家先看到8月说说可能会困惑。

### 3.5 陈默开始帮机房老师的时间矛盾

| 来源 | 时间 |
|------|------|
| 时间线.md | "10月初 - 陈默开始帮机房老师维护系统" |
| 群聊数据 | 2015年9月12日陈默说"机房老师让我帮忙维护学校的电脑系统" |

**结论**：群聊数据比时间线.md 早了约3周。需要统一。

### 3.6 夏建国去世时间细节

| 来源 | 描述 |
|------|------|
| 时间线.md | "2025年12月底 - 夏建国去世" |
| school_obituary_2025.json | "于2025年12月30日18时20分...逝世" |
| 时间线.md | "2025年12月30日 - 夏建国写邮件草稿给陈默（未发送）" |

**结论**：时间线.md 说12月30日夏建国写邮件草稿，但讣告说他12月30日18:20去世。如果他当天去世，写邮件草稿的时间应该在18:20之前，逻辑上可行但很紧张。

### 3.7 夏建国工作起始年份

| 来源 | 描述 |
|------|------|
| 人物.md | 1965年出生 |
| school_obituary_2025.json | "1985年参加工作" |

**结论**：1965年出生、1985年参加工作意味着20岁开始工作，合理（大专毕业）。无矛盾。

### 3.8 陈默奶奶疾病描述不一致

| 文档 | 2015年疾病 |
|------|-----------|
| 人物.md | "糖尿病并发症"，需手术费3万 |
| 群聊数据（2014年11月） | "高血压，最近要定期检查" |

**结论**：2014年说高血压，2015年变成糖尿病并发症。可以解释为病情发展，但群聊中应该有从高血压到糖尿病的过渡描述，否则显得突兀。

### 3.9 解密建议2.0 的谜题顺序与解密.md 不同

解密建议2.0 的认知节点流程：

```
父亲便签 → QQ登录 → 群聊 → QQ空间 → 光之公式 → 给小灯文件夹 → 父亲第一层日志 → 加密相册 → 隐写文字 → 加密日志 → 陈默动机 → 父亲第二层 → 完整信件 → 完成
```

解密.md 的谜题顺序：

```
阶段一（公开浏览）→ 谜题1（相册EXIF）→ 谜题2（父亲第一层）→ 谜题3（加密日志隐写）→ 谜题4（现代诗）→ 谜题5（父亲第二层）
```

**结论**：解密建议2.0 把父亲第一层放在相册之前（节点2.2 vs 2.4），而解密.md 把相册放在父亲第一层之前（谜题1 vs 谜题2）。顺序不同会影响玩家的认知递进和红鲱鱼效果。需要确定最终顺序。

## 四、总结

| 严重程度 | 数量 | 关键问题 |
|---------|------|---------|
| 阻断性 | 5个 | 人数矛盾、金额矛盾、父亲文件夹结构矛盾、项目现状密码错误、登录密码矛盾 |
| 混淆性 | 9个 | 时长矛盾、阶段时长矛盾、邮件触发/地址矛盾、红鲱鱼编号、QQ个人说明、职业、报酬、日志日期 |
| 细节性 | 9个 | 年级、张雨年级、光之公式时间、机房时间、去世时间、疾病描述、谜题顺序等 |

### 优先修复建议

1. **立即统一**：高考移民人数（18人）、涉案金额（2000万）、父亲文件夹密码体系
2. **尽快统一**：项目现状.md 的密码列表、登录密码设计、QQ个人说明（影响谜题）
3. **逐步修复**：红鲱鱼编号、邮件地址、职业描述、时长分配等
