# task-07：谜题5——回收站系统日志（被删除的不一定消失）

谜题编号：5
难度：★★★☆☆
所属阶段：阶段三（真相反转）
预计玩家用时：10-15分钟

---

## 谜题概述

林晓宇在陈默操作学籍数据后，截取了系统日志，悄悄保存在夏建国的电脑上。这个文件后来被删进了回收站，但从未被清空。玩家在日志中看到提示后，打开回收站找到这份技术证据。

无需密码，操作即解锁。

---

## 触发条件

谜题3（加密日志）解锁后，玩家在日志正文中看到：

```
我把那天晚上的系统日志截图保存了一份，放在夏叔叔的电脑上（他不知道）。
那是我唯一能用的电脑——我自己的电脑被陈默警告过，我不敢存在上面。
如果有一天我出事了，去回收站找找。
被删除的东西，不一定真的消失了。
```

调查笔记自动弹出：

```
晓宇的日志里提到他"保存了一份备份"。
备份在哪里？
去回收站找找。
```

---

## 回收站文件

文件名：`截图_1202.bmp`

状态：已删除（在回收站中）

文件名设计说明：文件名故意模糊（不直接写"系统日志"），让玩家通过查看文件内容和属性来判断其重要性。

---

## 文件属性（时间戳）

玩家右键文件 → 属性，可以看到：

| 字段 | 值 |
|------|-----|
| 创建时间 | 2015年12月02日 03:25 |
| 修改时间 | 2015年12月02日 03:25 |

注意：此处**不显示删除时间**。删除时间（2016年2月16日上午9:12，林晓宇坠楼后第二天）属于阶段四揭露的内容，由夏建国在《给夏灯》中自己坦白。

时间戳的意义：凌晨3:17陈默开始操作，3:23操作完成，3:25林晓宇截图——时间完全吻合。

---

## 文件内容：系统日志截图

`截图_1202.bmp` 是一张截图图片，显示以下系统日志内容：

```
[2015-12-02 03:17:43] 用户 sys_operator_x 登录
[2015-12-02 03:18:02] 执行操作：UPDATE student_records SET...
[2015-12-02 03:19:55] 修改学籍记录 #4821, #4822, #4823
[2015-12-02 03:21:30] 修改学籍记录 #4824, #4825, #4826
[2015-12-02 03:23:11] 操作完成，共修改6条记录
[2015-12-02 03:23:15] 用户 sys_operator_x 登出
```

---

## 叙事背景说明

林晓宇为什么把文件存在夏建国的电脑上：

日志正文中已解释："那是我唯一能用的电脑——我自己的电脑被陈默警告过，我不敢存在上面。"

这个细节同时揭示：
- 陈默曾警告林晓宇（两人之间的张力）
- 林晓宇信任夏建国（或者说，他没有更好的选择）
- 林晓宇在夏建国不知情的情况下使用了他的电脑

---

## 解密路径

1. 玩家在日志正文中看到"去回收站找找"
2. 打开桌面上的"回收站"图标
3. 发现被删除的文件：`截图_1202.bmp`
4. 双击查看文件内容（系统日志截图）
5. 右键文件 → 属性，查看时间戳：创建时间 2015年12月02日 03:25
6. 玩家意识到：凌晨3:17陈默操作，3:25林晓宇截图——时间完全吻合

---

## 情感反应

陈默的罪行被技术性地证实了——系统日志是最诚实的证人。

6条学籍记录，6个被挤占的名额，6个被改变命运的孩子。

玩家此时对陈默的情感是复杂的：他确实做了，但他是被迫的。

调查笔记弹出，夏灯写下：

```
sys_operator_x。
陈默。
6条记录。
凌晨3点17分。

他真的做了。
但他为什么要做？
```

---

## 与阶段四的呼应

玩家在阶段三发现了这个截图文件，但不知道是谁删除的，也不知道删除时间。

阶段四，玩家读到《给夏灯》，夏建国自己坦白：

```
林晓宇死后第二天，我在他的电脑上发现了一个截图文件。
我知道它意味着什么。
2月16日上午，我把它删了。
我告诉自己这是为了保护你，但我知道，这只是借口。
```

这个呼应让阶段四的情感冲击更深：玩家在阶段三发现了那个截图，现在才知道是父亲删除的。
