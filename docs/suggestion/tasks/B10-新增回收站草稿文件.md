# 新增回收站草稿文件

**类别**：B类
**优先级**：中
**依赖**：无
**修改文件**：`src/data/recycle_bin/`（新建2个文件）

## 背景

回收站当前只有1个条目，内容单薄。回收站是 Windows XP 最具象征意义的功能——被删除的不一定消失。两个草稿文件分别展示林晓宇曾经想过放弃、夏父曾经想过说出真相，但他们都没有，是整个故事"沉默"主题的具象化体现。

## 具体修改内容

在 `src/data/recycle_bin/` 目录下新建以下2个文件（或在 index.json 中添加对应条目）：

**文件1：举报信草稿.json**
```json
{
  "filename": "举报信草稿.txt",
  "deleted_by": "林晓宇",
  "deleted_at": "2015-12-05",
  "content": "县公安局：\n我是县一中高一学生，我发现照相馆有人在为外省学生伪造证件照...\n[文件在此处截断，后续内容已损坏]"
}
```

**文件2：给夏灯（草稿）.json**
```json
{
  "filename": "给夏灯（草稿）.txt",
  "deleted_by": "夏建国",
  "deleted_at": "2016-02-16",
  "content": "小灯，\n我知道林晓宇的死不是意外。\n我知道是谁做的。\n我知道我应该说出来。\n但是...\n[文件在此处截断]"
}
```

注意：夏父草稿的删除时间是2016年2月16日，即林晓宇坠楼后第二天。这个时间点说明夏父在林晓宇死后曾经想过说出真相，但最终选择了沉默。

## 验收标准

- 回收站中包含上述2个文件（或 index.json 中包含对应条目）
- 举报信草稿标注删除者为林晓宇，删除时间为2015年12月5日
- 夏父草稿标注删除者为夏建国，删除时间为2016年2月16日
- 两个文件内容均在关键处截断，保留悬念
- 文件在游戏中可通过回收站界面查看
