# 统一陈默2026年职业描述为IT运维工程师

**类别**：A类

**依赖**：无
**修改文件**：
- `docs/大纲.md`（"成为深圳程序员"处）

- `docs/解密.md`（新闻报道标题"27岁程序员自首"处）

## 背景

人物.md 明确定义陈默2026年职业为"深圳某公司IT运维工程师（非开发岗位）"，但大纲.md、时间线.md、解密.md 新闻报道标题均使用"程序员"。这一矛盾会导致开发者在制作新闻报道等素材时使用错误的职业描述，影响人物塑造的一致性（IT运维而非开发岗位，强调陈默是技术执行者而非创造者，更符合其被动参与的人物定位）。

## 具体修改内容

1. `docs/大纲.md`：将"成为深圳程序员"修改为"成为深圳IT运维工程师"
2. `docs/时间线.md`：将"在深圳做程序员"修改为"在深圳做IT运维"
3. `docs/解密.md` 新闻报道标题：将"27岁程序员自首"修改为"27岁IT运维工程师自首"

以人物.md 为权威来源，其他文档向人物.md 对齐。

## 验收标准

- 大纲.md、时间线.md、解密.md 中陈默职业描述均为"IT运维"或"IT运维工程师"
- 三处文档中不再出现将陈默描述为"程序员"的表述
- 人物.md 中的定义保持不变
