# 修改《给夏灯.txt》结尾增加灯的意象闭环

**类别**：C类
**优先级**：高
**依赖**：依赖 A03（README.txt 修改后，"灯熄灭的日子"才能成为密码的情感来源）
**修改文件**：`src/data/documents/给夏灯.txt`（或 src/data/documents/ 下对应的父亲信文件）

## 背景

密码"20151123"与"灯熄灭的日子"需要在《给夏灯.txt》的结尾形成完整的叙事闭环。夏父用女儿的名字命名了自己良知的熄灭，又用这个日期作为密码，把最后的真相留给了女儿。这个循环是整个游戏最精密的叙事设计之一，当前版本结尾缺少这个闭环。

## 具体修改内容

在《给夏灯.txt》的结尾（最后一段之后）添加以下内容：

```
小灯，我给你起这个名字，是希望你能照亮黑暗。
但我自己的灯，在2015年11月23日就熄灭了。

你的灯还亮着。
这是我唯一感到欣慰的事。

爸爸
```

同时，在信的第一部分（描述第一次签字那一段）之后，添加以下语气断裂的段落，增加真实感：

```
我不知道为什么要写这些细节。
也许是因为我想让你知道，我不是一个坏人。
但我做了坏事。
这两件事可以同时是真的。我花了很多年才接受这一点。
```

在信的第四部分，在"我希望你能照亮黑暗"之前，添加：

```
小灯，我要告诉你一件我一直不敢说的事。
2015年11月23日，校长说"你女儿今年高一了，成绩不错"的时候，我想到了你。
然后我签了字。
我告诉自己，这是为了你。
但这是谎言。
真相是：我用你当借口。我用对你的爱，来掩盖我的恐惧。
这是我对你做的最坏的一件事。
```

## 验收标准

- 《给夏灯.txt》结尾包含"我自己的灯，在2015年11月23日就熄灭了"
- 结尾包含"你的灯还亮着。这是我唯一感到欣慰的事。"
- 包含"我用你当借口"的自我揭露段落
- 包含第一次签字后的语气断裂段落
- 玩家读到结尾时，会想起密码"20151123"和"灯熄灭的日子"，形成叙事闭环
