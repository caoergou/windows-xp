# 《山月无声》游戏设计

*素材规格与叙事控制*

---

## 设计概览

### 游戏形式

模拟Windows XP桌面环境，玩家通过以下方式调查真相：

- 本地文件系统（文件管理器）
- QQ客户端（群聊、好友、空间）
- 模拟浏览器（贴吧存档、网页快照）
- 邮件客户端（与陈默往来）

### 时间跨度

- **过去线**：2015-2016年（通过数字遗迹呈现）
- **现在线**：2026年1-2月（玩家操作时间）

### 游戏时长

3小时（180分钟）

- 解密：75分钟
- 阅读：105分钟

---

## 素材组织结构

### 素材分类

```
assets/
├── chat-logs/          # 群聊记录（JSON）
├── qq-space/           # QQ空间内容（JSON）
├── emails/             # 陈默邮件（JSON）
├── documents/          # 本地文档（TXT + JSON元数据）
├── photos/             # 照片（JPG + JSON EXIF）
├── web-archives/       # 网页存档（JSON）
└── 清单.md             # 素材制作清单
```

### 数据格式规范

所有结构化数据使用JSON格式：

- 群聊记录：JSON数组
- QQ空间：JSON对象
- 邮件：JSON数组
- 照片EXIF：JSON元数据文件
- 网页存档：JSON结构

纯文本内容（父亲日志、林晓宇诗歌）：TXT格式

### 3. 邮件

**文件**：`assets/emails/chenmo-emails.json`

**数据结构**：

```json
{
  "correspondence": [
    {
      "id": "email_001",
      "from": "chenmo.dev@gmail.com",
      "to": "xiadengnj@163.com",
      "date": "2026.01.05",
      "subject": "惊闻噩耗",
      "body": "...",
      "trigger": "game_start",
      "status": "unread"
    }
  ]
}
```

**触发机制**：

| 邮件 | 触发条件 | 情感状态 |
|------|---------|---------|
| 邮件1 | 游戏开始 | 慰问、试探 |
| 邮件2 | 玩家回复邮件1 | 回避、引导 |
| 邮件3 | 破解相册后 | 痛苦、坦白部分 |
| 邮件4 | 看完父亲日志后 | 承认在现场 |
| 邮件5 | 看完短信完整版后 | 完整证词 |
| 邮件6 | 读完所有证据 | 决心、告别 |

**自动交互机制**：
- **自动草稿**：当玩家在游戏中解锁关键证据（如破解相册、看完日记）后，打开邮件客户端，系统会自动根据当前进度生成一封发给陈默的草稿。
- **一键发送**：玩家点击“发送”即可。
- **触发回复**：发送后，经过短暂的模拟延迟（或立即），收到陈默的回复，推动剧情进入下一阶段。

**设计原则**：

- 邮件只提供方向性引导，不直接给密码
- 情感递进：试探→回避→痛苦→坦白→告别
- 不发送证据包附件

### 关键词过滤规则

**前期（阶段一、二）禁用词**：

- "高考移民"
- "15人"
- "三年"
- "250万"
- "产业链"
- "保护伞"
- 受害者完整故事

**替代表述**：

- 用"一些事情"、"某个秘密"代替"替考案"
- 用"几个学生"代替"6人"、"18人"
- 用"不法交易"代替"高考移民操作"
