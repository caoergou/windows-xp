## 🤖 一、智能体角色定义

### 1. 项目协调者（Project Coordinator）

**角色定位**：总指挥和协调者

**核心职责**：
- 制定整体规划和任务分配
- 协调各个智能体的工作进度
- 审核和整合各智能体的输出
- 解决冲突和优先级问题
- 进行最终质量把关

**工作方式**：
- 与用户直接沟通
- 创建和维护任务看板
- 定期检查各智能体进度
- 合并代码和内容

**所需技能**：
- 项目管理能力
- 全局视角
- 代码审查能力
- 内容质量把控

## 📋 二、工作流程设计

```
Day 1: 项目启动
├─ 项目协调者：召开启动会议，分配任务

├─ 数据架构师：完成数据结构设计，创建模板
├─ 前端开发：完成密码输入组件，开始加密文件夹
```

## 🔄 三、协作机制

### 3.1 通信方式

**文件系统协作**：
- 所有智能体共享同一个代码仓库
- 使用Git进行版本控制
- 每个智能体在独立分支工作

**任务看板**：
- 使用 `docs/任务看板.md` 追踪进度
- 每个任务标注：负责人、状态、优先级
- 每日更新进度

**文档共享**：
- 数据规范文档：`docs/数据规范.md`
- 人物设定文档：`docs/人物.md`
- 时间线文档：`docs/时间线.md`

---

### 3.2 依赖关系

```
数据架构师 → 内容创作智能体
（必须先有数据结构，才能创作内容）

前端开发 → 测试与质量保证
（必须先有功能实现，才能测试）

内容创作智能体 → 项目协调者
（内容完成后，由协调者整合）

项目协调者 → 测试与质量保证
（整合完成后，才能全面测试）
```

---

### 3.3 冲突解决

**优先级规则**：
1. 阻塞性问题优先（影响其他智能体工作）
2. 核心功能优先（解密系统、林晓宇内容）
3. 补充功能其次（UI优化、进度追踪）

**决策机制**：
- 技术问题：前端开发 + 项目协调者决策
- 内容问题：相关内容创作智能体 + 项目协调者决策
- 优先级冲突：项目协调者最终决策

## 🚀 五、如何启动多智能体协作

### 方式一：使用Claude Code多会话（推荐）

1. **打开多个Claude Code会话**
   - 每个会话分配一个智能体角色
   - 在会话开始时明确告知角色定位

2. **共享工作目录**
   - 所有会话指向同一个项目目录
   - 使用Git管理版本

3. **协调机制**
   - 主会话（项目协调者）负责分配任务
   - 其他会话完成后通知主会话
   - 主会话负责合并和审核

### 方式二：使用Task工具并行启动

```javascript
// 项目协调者可以这样启动多个智能体
Task({
  subagent_type: "general-purpose",
  description: "前端开发智能体",
  prompt: "你是前端开发智能体，负责实现解密系统..."
})

Task({
  subagent_type: "general-purpose",
  description: "内容创作-林晓宇",
  prompt: "你是内容创作智能体，专门负责林晓宇的所有内容..."
})
```

### 方式三：顺序启动（适合小团队）

1. 先启动数据架构师，完成数据结构
2. 再启动前端开发和内容创作智能体并行工作
3. 最后启动测试智能体

## 💡 七、协作最佳实践

### 7.1 命名规范

- `feature/frontend-password-dialog` - 前端功能
- `content/linxiaoyu-shuoshuo` - 内容创作
